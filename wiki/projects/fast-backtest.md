# fast-backtest

Numba-accelerated grid trading backtest engine.
Drop-in replacement for scalping-pnl, compatible with its config format.

## Speed (145k bars, BTCUSDT)

| Mode | Simulation | Total (load + sim + write) |
|------|-----------|---------------------------|
| Pure Python (3.13) | ~1050 ms | ~8 s |
| Numba JIT, 1st call | ~500 ms | ~7 s |
| Numba JIT, 2nd+ call | **~7 ms** | ~6 s |
| **1579 configs (batch)** | **~11 s** | **~3–5 min** |

---

## Quick Start (Docker)

### 1. One-command batch run (interactive)

```bash
./run_batch.sh
# → prompts for configs dir and results dir
# → builds image on first run (~5 min for Numba compile)
# → processes all *.json configs, writes results/batch_summary.csv
```

### 2. With explicit paths

```bash
./run_batch.sh /path/to/my/configs /path/to/my/results
```

### 3. Full output (equity_curve, trade logs, pnl per run)

```bash
FULL_OUTPUT=1 ./run_batch.sh /path/to/configs /path/to/results
```

### 4. Resume interrupted run

```bash
./run_batch.sh --resume /path/to/configs /path/to/results
```

---

## Quick Start (Local Python)

Requires Python 3.9–3.12 (Numba doesn't support 3.13).

```bash
python3.9 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt

# Single config
python batch_run.py --configs 1580_backtest_configs --results results

# Or with explicit paths
python batch_run.py \
  --configs /Users/andriy/VisualStudio/fast-backtest/1580_backtest_configs \
  --results /Users/andriy/data/results
```
